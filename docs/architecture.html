Architecture & Setup

Full developer reference for building on top of Catylst.

Overview

Catylst is a fully-wired KMP project with Android, iOS, and Desktop targets. Every built-in feature can be stripped in minutes using the included Claude Code skills.

Built-in features: Compose Multiplatform UI, Koin DI, Room 3 database, Ktor networking, Navigation3, WorkManager notifications, cross-platform permissions, multiplatform preferences, and provider-agnostic AI (Claude / Groq / Gemini).

Project Setup

Run the setup script once immediately after cloning. It renames everything - package, app name, source directories, iOS config - then deletes itself and the docs/ folder.

# Clone
git clone https://github.com/your-org/catylst.git myapp
cd myapp

# Rename (run once - self-destructs after)
bash scripts/setup.sh com.yourname.yourapp YourApp
StepWhat happens
1Replace io.jadu.catylst in every .kt, .kts, .xml, .plist, .pbxproj, .xcconfig, .swift
2Replace Catylst app name in string resources, manifests, iOS xcconfig
3Update rootProject.name in settings.gradle.kts
4Rename source directory trees across all source sets
5Delete docs/ folder
6Delete itself
One-time only. After it runs, sync Gradle in your IDE then build once to verify.

After setup

./gradlew :androidApp:assembleDebug

AI Services Setup

Three providers included. Swap with one line in AppModule.kt.

1. Add your API key

cp local.properties.example local.properties
# local.properties - git-ignored, never commit
claude.api.key=sk-ant-...
groq.api.key=gsk_...
gemini.api.key=AIza...

// di/AppModule.kt - activate exactly one line
single<AiProvider> { ClaudeProvider(get(), AppConfig.claudeApiKey) }

// single<AiProvider> { GeminiProvider(get(), AppConfig.geminiApiKey) }

Module Structure

YourApp/
├── androidApp/               # Android entry point only
│   └── src/main/kotlin/.../
│       └── MainActivity.kt   # Injects API keys, starts Koin
│
└── composeApp/               # ALL shared code
    └── src/
        ├── commonMain/kotlin/.../
        │   ├── ai/               # AiProvider + Claude/Groq/Gemini
        │   ├── config/           # AppConfig (API key holder)
        │   ├── data/
        │   │   ├── local/        # Room entities + DAOs
        │   │   ├── preferences/  # AppPreferences
        │   │   └── repository/
        │   ├── di/               # AppModule.kt - all Koin bindings
        │   ├── navigation/       # Screen.kt + AppNavigation.kt
        │   ├── network/          # HttpClientFactory + ApiService
        │   ├── notifications/    # expect NotificationScheduler
        │   ├── permissions/      # expect PermissionController
        │   └── ui/
        │       ├── screens/
        │       └── viewmodel/
        ├── androidMain/      # actual implementations for Android
        ├── iosMain/          # actual implementations for iOS
        └── desktopMain/      # actual implementations for Desktop
Rule: Put everything in commonMain. Only use platform source sets for actual implementations of expect declarations.

Navigation

Navigation3 with rememberNavBackStack() and Crossfade. Never use NavDisplay - it is not KMP-compatible.

Adding a screen

// 1. Screen.kt
@Serializable
data object Xxx : Screen()

// 2. AppNavigation.kt - SerializersModule
subclass(Screen.Xxx::class)

// 3. AppNavigation.kt - Crossfade branch
is Screen.Xxx -> XxxScreen(onBack = { backStack.removeLastOrNull() })

// 4. HomeScreen.kt - button
OutlinedButton(onClick = onNavigateToXxx) { Text("Open Xxx") }
Use the kmp-add-screen Claude Code skill for the full guided walkthrough.

Common Pitfalls

BuildConfig unresolved in composeApp
Lives in :androidApp only. Use AppConfig object in commonMain instead.
NavDisplay not found
Not KMP-compatible. Use Crossfade + rememberNavBackStack() as in AppNavigation.kt.
java.util.Properties in defaultConfig {}
Declare Properties at .kts file top-level, not inside defaultConfig block.
Room entity changes not picked up
Run ./gradlew :composeApp:kspAndroidMain after any Entity/DAO change.
Koin inject fails at runtime
Check binding exists in appModule() or platformModule().
buildConfigField value is empty
Use "\"${value}\"" format in .kts - bare quotes get stripped.
Inline qualified import
Add import at file top, use unqualified name in body.
KN 2.x ObjC method not found (iOS)
Use ClassName.Companion.method() instead of top-level call.
