Setup State Check

Before writing any code, check if scripts/setup.sh exists. If it does, the template has not been initialized - ask the user to run it first. If it does not exist, proceed normally using the package name found in existing .kt files.

# Setup NOT run - ask user to execute:
bash scripts/setup.sh com.yourname.yourapp YourApp

# Setup complete - proceed normally
Never write code using io.jadu.catylst as the package. Always check the actual package in existing .kt files first.

Module Structure

androidApp/              # Android entry point ONLY - MainActivity, BuildConfig
composeApp/              # ALL shared code lives here
  src/
    commonMain/          # Platform-agnostic Kotlin + Compose - put everything here
    androidMain/         # actual implementations for Android
    iosMain/             # actual implementations for iOS (Kotlin/Native)
    desktopMain/         # actual implementations for Desktop JVM
Rule: Put everything in commonMain. Only use platform source sets for actual implementations of expect declarations, or APIs that genuinely cannot be shared.

Key Files

FileRole
composeApp/.../di/AppModule.ktAll Koin bindings - start here when adding features or debugging DI
composeApp/.../di/PlatformModule.ktexpect declaration, actual per source set (NotificationScheduler, ObservableSettings, etc.)

composeApp/build.gradle.ktsShared module - all KMP dependencies, KSP, Room schema config
gradle/libs.versions.tomlVersion catalog - single source of truth for all dependency versions

Build Commands

# Standard builds
./gradlew :androidApp:assembleDebug                        # Android debug APK
./gradlew :androidApp:assembleRelease                      # Android release APK
./gradlew :composeApp:compileKotlinIosSimulatorArm64       # iOS compile check (no Xcode needed)
./gradlew :composeApp:run                                  # Desktop

# Tests
./gradlew :composeApp:test                                 # Common (KMP) unit tests
./gradlew :composeApp:testDebugUnitTest                    # Android unit tests

# Room - run after any Entity or DAO change
./gradlew :composeApp:kspAndroidMain

# Convenience scripts
bash scripts/build.sh          # Android + iOS compile
bash scripts/test.sh           # All unit tests

Critical Notes

Navigation: Never use NavDisplay - it is not KMP-compatible. Always use rememberNavBackStack() + Crossfade as shown in AppNavigation.kt.
BuildConfig: BuildConfig lives in :androidApp only and cannot be accessed from composeApp. Use AppConfig in commonMain instead.
Room changes: After adding or modifying any @Entity or @Dao, run ./gradlew :composeApp:kspAndroidMain to regenerate Room code before building.
iOS / KN 2.x: Kotlin/Native 2.x changed how Objective-C category methods are imported. If a top-level ObjC call fails, use ClassName.Companion.method() instead. Always verify iOS with ./gradlew :composeApp:compileKotlinIosSimulatorArm64 after touching iosMain.
buildConfigField: Always use "\"${value}\"" format in .kts files. Bare quotes get stripped by linters, producing an empty BuildConfig field.
Imports: Never use inline qualified names like android.content.Context.MODE_PRIVATE in code. Add the import at the file top and use the unqualified name in the body.

Common Pitfalls

BuildConfig unresolved in composeApp
Use AppConfig object in commonMain - BuildConfig is androidApp-only.
NavDisplay not found
Not KMP-compatible. Use Crossfade + rememberNavBackStack().
java.util.Properties in defaultConfig {}
Declare at .kts file top-level, not inside defaultConfig block.
Room entity changes not reflected
Run kspAndroidMain after any Entity/DAO change.
Koin runtime injection failure
Verify binding exists in appModule() or platformModule().
buildConfigField value is empty string
Use "\"${value}\"" - bare quotes get stripped.
KN 2.x ObjC category method missing
Use ClassName.Companion.method() instead of top-level call.
Inline qualified import in code body
Add import at file top, use unqualified name in body.
